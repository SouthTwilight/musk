## 二、Phase 1: 框架核心 MVP（详细任务分解）

### Sprint 1: 项目脚手架 + 认证系统（约 7 天）

| # | 任务 | 前端 | 后端 | 产出物 |
|---|------|------|------|--------|
| 1.1 | 初始化项目结构 | Vite 创建 Vue 3 + TS 项目 | Django 项目 + 虚拟环境 | 项目骨架 |
| 1.2 | 用户模型 | — | User 模型（is_staff=True 默认） | 数据库表 |
| 1.3 | 注册/登录 API | — | `/api/auth/register/`、`/api/auth/login/`、`/api/auth/refresh/` | REST API |
| 1.4 | 前端登录页 | 登录/注册表单 + 路由守卫 | — | 登录页面 |
| 1.5 | JWT 集成 | Axios 拦截器自动附带 Token | Simple JWT 配置 | 认证闭环 |
| 1.6 | 基础配置 | 环境变量管理 | settings.py + .env | 配置系统 |

**Sprint 1 验证标准**：用户可注册 → 登录 → 进入空白主页 → Token 过期可刷新

| 2.4 | 背景图系统 | 内容区背景图 + 遮罩叠加 | 文件上传接口 | 背景图功能 |

**Sprint 2 验证标准**：登录后看到完整布局 → 侧边栏可折叠 → 主题可切换 → 可上传背景图

### Sprint 3: 模块应用层 + AI 中枢（约 7 天）

| # | 任务 | 前端 | 后端 | 产出物 |
|---|------|------|------|--------|
| 3.1 | 自动扫描器 | — | `scanner.py` 遍历 apps/ 读 manifest | 模块自动发现 |
| 3.2 | 模块注册表 | 前端 `moduleRegistry` 动态注册路由/菜单 | — | 前端模块系统 |

| 3.4 | AI 模型适配 | — | DeepSeek + GLM 适配器 | AI 调用能力 |

## 六、开发规范

### 6.1 代码规范

- **Python**：PEP 8，Black 格式化，类型注解
- **TypeScript/Vue**：ESLint + Prettier，Composition API + `<script setup>`
- **Git Commit**：Conventional Commits 格式（中文化）
- **分支策略**：`main`（稳定） + `develop`（开发） + `feature/*`（功能分支）

### 6.2 测试策略

| 层级 | 范围 | 工具 |
|------|------|------|
| 后端单元测试 | Django Model/Service 层 | pytest + pytest-django |
| 后端 API 测试 | REST 端点行为 | pytest + DRF test client |
| 前端组件测试 | Vue 组件渲染和交互 | Vitest + Vue Test Utils |
| 端到端测试 | 关键用户流程 | （后期可选 Cypress） |

### 6.3 模块开发流程

```text
创建模块
    │
    ├── 1. 在 apps/ 下创建目录
    ├── 2. 编写 manifest.py
    ├── 3. 实现 models.py (独立 DB)
    ├── 4. 实现 views.py (业务逻辑)
    ├── 5. 编写 urls.py (路由)
    ├── 6. 前端模块目录 (路由 + 组件)
    ├── 7. 本地验证
    └── 8. 集成测试
```

## 七、风险与缓解

| 风险 | 发生概率 | 缓解措施 |
|------|----------|----------|
| 模块接口契约在首个模块开发时发现不足 | 中 | 博客引擎作为试金石模块，先做再推广 |
| AI API 兼容问题 | 低 | DeepSeek 兼容 OpenAI SDK，GLM 有官方适配 |
| Django 多数据库路由复杂 | 低 | 已有成熟的 DATABASE_ROUTERS 方案 |
| 前端动态路由注册遇到 Vue 限制 | 低 | Vue Router 4 原生支持 `addRoute()`，有明确模式 |
